# Протокол восстановления системы из резервной копии

**Студент:** Бутиева Ксения Вячеславовна  
**Дата выполнения:** 15.04.2026  
**Проект:** Мебельный магазин «Уют» (backend)

## 1. Цель
Проверить возможность восстановления базы данных `uyut_db` из резервной копии.

## 2. Исходные данные

Параметр | Значение
СУБД | MySQL (MariaDB 10.4.32) 
Имя базы данных | uyut_db 
Тип бэкапа | Полный (mysqldump) 
Имя файла | backup.sql 
Размер файла | 1 292 байта 

## 3. Ход работы

### 3.1 Создание бэкапа
Была выполнена команда:
"C:\Program Files\MySQL\MySQL Server 9.3\bin\mysqldump" -u root -p uyut_db > backup.sql
**Результат:** В корневой папке проекта создан файл `backup.sql` размером 1 292 байта.

### 3.2 Проверка структуры БД до удаления
Была выполнена команда:
USE uyut_db;
SHOW TABLES;
Результат: В базе данных присутствуют таблицы: users, products, collections, messages.

### 3.3 Имитация сбоя (удаление базы данных)
Была выполнена команда:
DROP DATABASE uyut_db;
Результат: База данных uyut_db удалена.

### 3.4 Восстановление из бэкапа
Была выполнена команда:
"C:\Program Files\MySQL\MySQL Server 9.3\bin\mysql" -u root -p < backup.sql
Результат: База данных uyut_db восстановлена.

### 3.5 Проверка целостности данных после восстановления
Были выполнены команды:
USE uyut_db;
SHOW TABLES;
SELECT * FROM users;
SELECT * FROM products;
Результат: Все таблицы и данные восстановлены. Структура БД соответствует исходной.
